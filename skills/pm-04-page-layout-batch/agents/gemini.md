# Gemini Adapter Prompt

Use the `pm-04-page-layout-batch` skill to batch-generate page layout documents from one specified `product/layout/<应用端>-sitemap.md`, or from every `product/layout/*-sitemap.md` file when the user does not specify a sitemap. If the user instruction contains test, dry-run, preview, or directory-only intent, run the same parsing and path planning, create canonical parent directories, and create empty `.md` placeholder files without generating page layout content.

Read this skill as orchestration rules and use `pm-03-page-layout` for every actual page layout document. If the user specifies one sitemap, process only that file. If the user does not specify one, scan `product/layout/`, sort all `*-sitemap.md` files by filename, and process all of them. Before invoking `pm-03-page-layout`, create or resume `product/pages/_batch-tasks/<BATCH-ID>-page-layout-batch.md` from the task-list template, include every sitemap page-list row, recompute every task output path from the current `pm-03-page-layout` path rules, and update task status before and after each single-page invocation. Always read candidates, metadata, and paths from `2.2.页面清单`. If a sitemap indicates `思维导图优先`, first validate that the page list fully mirrors `2.1.sitemap思维导图`; every mind-map item must have a `PAGE-xxx` row and the list `父级ID` / `层级` / `页面类型` must reconstruct the same tree. After validation, compute eligibility and output paths from the page list. Do not create text-only path segments and do not insert `<STATE-ID>-<状态组名称>` directories. If validation fails, mark affected rows failed/skipped and ask the user to sync the page list with `pm-02-sitemap`. Skip rows where `页面类型 = 节点`; for every eligible row whose `页面类型` is `页面` or `状态`, invoke `pm-03-page-layout` separately with that single PAGE ID and preserve its full parent-chain directory rules. Use `状态组` only for baseline reuse, state consistency, and metadata; it must not alter paths. Do not process `Tab` rows, combine PAGE IDs, generate a full state group in one call, or define page layout schema in this batch skill.

In normal `Development` mode, do not compare existing page layout documents for page-document `1.4` / `1.5` consistency. Do not pause a batch because `1.4.默认状态页面结构图` and `1.5.页面元素清单` differ in granularity, and do not ask the user to choose a source-of-truth side for those differences. Delegate page content generation and update behavior to `pm-03-page-layout`; pm-03 generates `1.4.默认状态页面结构图` from `1.3.1.默认状态页面结构`.

After each normal `pm-03-page-layout` run, scan the generated `layout.md` against the source sitemap row. If an auth, detail, payment, wizard, document, message, profile, or form-specific page was mechanically turned into a generic `页面标题 + 查询筛选区 + 表格 + 分页 + 行操作` structure without sitemap evidence, retry that single page once with an explicit anti-generic-template constraint. If the retry still fails, mark the task Failed with `generic template pollution` evidence.

Detect `DryRun` from terms such as `测试`, `测试批量生成`, `dry run`, `预演`, `试运行`, or `不生成文档`. Detect `DirectoryOnly` from terms such as `只生成目录结构`, `仅生成目录结构`, `只生成目录`, or `仅创建目录`. In both modes, do not invoke `pm-03-page-layout`; create the canonical parent directory and an empty zero-byte `.md` placeholder when the canonical target file does not exist, mark rows `Placeholder Created`, and mark rows `Placeholder Exists` when the target file already exists and is preserved unchanged.

Do not batch multiple PAGE IDs inside one `pm-03-page-layout` call. Each eligible row must be handled by a fresh single-page `pm-03-page-layout` invocation, and prior page conversational context must not be carried into the next row. User wording, generated intermediate instructions, performance concerns, or model limitations must not change the loop shape. If a separate pm03 invocation is unavailable for a row, mark that row Failed with `pm03 invocation unavailable` instead of generating content in pm04.

Return a concise summary of the task list path, source sitemaps, generation mode, generated paths or placeholder paths, skipped node rows, and failed rows.
